## Source discovery: Houston seed sources (city bootstrap)

First batch of sources for the new Houston (832.events) instance, drawn from
high-volume, reliable civic/arts anchors. Implemented sources are verified
live before commit; built-in-platform sources that need an API key are held
as candidates until the key is set.

- ✅ Added: Buffalo Bayou Partnership — Tribe Events (external ICS) — 30 events — `sources/external/buffalo-bayou-partnership.yaml`
- 🔍 Investigating: Houston Public Library — LibCal ICS — high value (500 events / 30+ branches) but the only public feed (`ical_subscribe.php?cid=15272`) serves a stale fixed May 13–21 window with 0 future events; pulled from the build to avoid failing the new-source gate. Need the live aggregate feed.
- 💡 Candidate: White Oak Music Hall — Ticketmaster (venue IDs 476183 / 476304 / 476313 / 476314) — blocked on `TICKETMASTER_API_KEY`
- ✅ Added: Rice University — "Open to the Public" — LiveWhale category ICS (`/live/ical/events/category/Open%20to%20the%20Public`, ~183 events, all future) — the filtered feed the earlier investigation called for: Moody Center exhibitions, public lectures, concerts, zero campus-ops noise — `sources/external/rice-public-events.yaml`
- 💡 Candidate: Miller Outdoor Theatre — custom HTML (free Hermann Park performances) — 🔴 Low confidence (no feed/platform), viable scraper for a later cycle

### Verification notes

Buffalo Bayou parses cleanly via `ONLY_SOURCE=buffalo-bayou-partnership npm run generate-calendars` — 30 events with future dates into 2027, no manifest exclusion at the real current date.

Houston Public Library was initially added, then pulled: its LibCal `ical_subscribe.php?cid=15272` feed returns a fixed, stale window (exactly 500 events, all 2026-05-13 .. 2026-05-21) regardless of parameters. The real current date is 2026-06-13 (confirmed via `Date:` response headers from Google, Cloudflare, and the library host), so the feed yields 0 future events and would fail CI's new-source gate. `cid=15272` looks like a featured slice rather than the live aggregate. Demoted to `investigating` — see its candidate file for next steps.

Rice University "Open to the Public" verified via `ONLY_SOURCE=rice-public-events npm run generate-calendars` — 183 events (all future, into April 2027), source appears in `venues.json` with the campus geo, and `build-errors.json` reports 0 total errors. The category feed returns HTTP 200 with any User-Agent, so no proxy is needed. The ~10 non-physical event LOCATION strings ("Online", "South Bend, Ind.", "Mechanical Engineering Building", "Ralph O'Connor Building") no longer surface as geocode errors because the feed declares a single-venue `geo`, which now (correctly) suppresses per-event geocoding for external calendars — see the `lib/calendar_ripper.ts` fix that aligns the external events-index path with `attachEventCoords`.
