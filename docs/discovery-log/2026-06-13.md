## Source discovery: Houston seed sources (city bootstrap)

First batch of sources for the new Houston (713.events) instance, drawn from
high-volume, reliable civic/arts anchors. Implemented sources are verified
live before commit; built-in-platform sources that need an API key are held
as candidates until the key is set.

- ✅ Added: Houston Public Library — LibCal (external ICS) — 500 events — `sources/external/houston-public-library.yaml`
- ✅ Added: Buffalo Bayou Partnership — Tribe Events (external ICS) — 30 events — `sources/external/buffalo-bayou-partnership.yaml`
- 💡 Candidate: White Oak Music Hall — Ticketmaster (venue IDs 476183 / 476304 / 476313 / 476314) — blocked on `TICKETMASTER_API_KEY`
- 🔍 Investigating: Rice University Events — LiveWhale ICS (`/live/ical/events`, 702 events) — all-events feed too noisy (campus-ops + out-of-town entries); need a filtered group/tag feed
- 💡 Candidate: Miller Outdoor Theatre — custom HTML (free Hermann Park performances) — 🔴 Low confidence (no feed/platform), viable scraper for a later cycle

### Verification notes

Both implemented feeds parse cleanly via `ONLY_SOURCE=houston-public-library,buffalo-bayou-partnership npm run generate-calendars`. The HPL feed serves "upcoming from the server's today, capped at 500"; because HPL runs 50+ events/day those 500 only span about a week. The dev sandbox clock ran ~1 month ahead of the live world, so an unmodified local build saw all 500 as past; re-running under `faketime '2026-05-13'` (aligned with the live feed window) produced the full 500 future events. CI runs on real-world UTC, aligned with the feed, so HPL populates normally there.
